# 棋谱导入功能实现文档

## 概述

本次更新实现了完整的棋谱导入与回放功能，支持多种棋谱格式的解析和着法动画回放。

## 功能特性

### 1. 支持的棋谱格式

- **XQF 格式**：标准中国象棋棋谱格式
- **纯文本棋谱**：支持中文着法记录
- **FEN 格式**：支持残局初始局面

### 2. 中文着法解析

支持标准中文着法格式：
- 基本格式：`棋子名 + 列号 + 方向 + 目标`
- 例如：`炮二平五`、`马八进七`、`车一进三`

支持特殊着法：
- 前缀标记：`前车进三`、`后马退五`
- 黑方着法：`马8进7`、`车9平8`

### 3. 着法动画回放

- 支持逐步前进/后退
- 支持跳转到任意着法
- 支持键盘快捷键（左右箭头）
- 吃子动画效果

### 4. 棋谱信息显示

在右下方面板显示：
- 标题
- 赛事
- 结果
- 来源
- 着法数

## 着法解析算法

### 1. 基本着法解析

```
输入: "炮二平五"
解析:
  - 棋子: 炮 (PIECES.R_CANNON)
  - 列号: 二 → 红方第2列 → 内部列号 7
  - 方向: 平 → 横向移动
  - 目标: 五 → 红方第5列 → 内部列号 4
输出: { pieceType, fromCol, direction, targetValue }
```

### 2. 特殊着法解析

```
输入: "前车进三"
解析:
  - 前缀: 前 → 选择前面的棋子
  - 棋子: 车
  - 方向: 进 → 向前移动
  - 步数: 三 → 移动3格
处理:
  1. 找到该列所有车
  2. 按位置排序（前/后）
  3. 选择对应的棋子
  4. 计算目标位置
```

### 3. 坐标系统

```
内部坐标:
  - 行: 0=黑方底线, 9=红方底线
  - 列: 0=最左, 8=最右

红方列号转换:
  - 中文: 一二三四五六七八九
  - 内部: 8,7,6,5,4,3,2,1,0

黑方列号转换:
  - 数字: 123456789
  - 内部: 0,1,2,3,4,5,6,7,8
```

## 已知限制

1. **着法解析失败**：某些非标准格式的着法可能无法解析
2. **棋谱验证**：导入时不会验证着法是否合法
3. **残局支持**：FEN 格式支持有限

## 错误处理

### 解析失败处理

```typescript
// 解析失败时跳过该着法并记录错误
if (!parsed) {
  console.error(`❌ 着法解析失败: ${move.red}`);
  continue; // 跳过
}
```

### 回合号不连续检测

```typescript
// 检测缺失的回合
if (round !== lastRound + 1) {
  console.error(`❌ 回合号不连续: 期望 ${lastRound + 1}，实际 ${round}`);
}
```

## 使用方法

### 1. 导入棋谱

1. 点击"导入棋谱"按钮
2. 选择棋谱文件或粘贴内容
3. 点击"导入并开局"

### 2. 回放着法

- **键盘**：左右箭头键
- **鼠标**：点击着法列表
- **按钮**：前进/后退按钮

### 3. 研究模式

导入后自动进入研究模式：
- 可以自由移动棋子
- 不会记录新着法
- 点击"新游戏"退出
