## 🔍 调试信息

### 关键日志说明

#### 1. 点击检测日志
```javascript
点击检测: {
  pieceColor: 'red',      // 被点击棋子的颜色
  currentPlayer: 'red',   // 当前行棋方
  isCurrentPlayer: true   // 是否是己方棋子
}
```

**判断逻辑**:
- `isCurrentPlayer === true` → 允许点击
- `isCurrentPlayer === false` → 阻止点击

---

#### 2. 阻止点击日志
```javascript
❌ 不能点击对方棋子 (black)
```

**触发条件**: `pieceColor !== chessStore.currentPlayer`

---

#### 3. AI 思考状态日志
```javascript
onMouseDown - AI思考状态: true
AI 正在思考中，禁止操作
```

**触发条件**: `isAIThinking() === true`

## 🐛 常见问题排查

### 问题1: 仍然可以点击对方棋子

**可能原因**:
1. `chessStore.currentPlayer` 的值不正确
2. `pieceColor` 的判断逻辑有误

**排查步骤**:
1. 查看控制台输出的 `点击检测` 日志
2. 确认 `pieceColor` 和 `currentPlayer` 的值
3. 检查是否输出了 `❌ 不能点击对方棋子` 日志

**解决方案**:
- 如果 `currentPlayer` 值错误，检查 store 初始化
- 如果 `pieceColor` 值错误，检查棋子 userData 中的 piece 值

---

### 问题2: AI 思考时仍可以点击棋子

**可能原因**:
1. `isAIThinking()` 回调返回 `false`
2. AI 模块未正确初始化

**排查步骤**:
1. 查看 `onMouseDown - AI思考状态:` 的日志
2. 确认是否为 `true`

**解决方案**:
- 检查 [aiModule](file://v:\4_mydoc\tauri\nicechchess\src\components\3d\ChessBoard3D.vue#L60-L60) 是否正确初始化
- 检查 [isAIThinking.value](file://v:\4_mydoc\tauri\nicechchess\src\components\3d\useAI.ts#L18-L18) 是否在 AI 思考时设置为 `true`
